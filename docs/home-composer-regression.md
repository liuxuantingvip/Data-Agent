# 首页输入框回归用例

## 范围

- 页面：`/`
- 目标：首页输入框、`@` 工具唤醒、内联 tool token、首页卡片区解耦、弹层不回归破坏

## 用例

- `TC01` 空态进入编辑
  - 初始展示占位提示
  - 点击输入框空白区后进入编辑态
  - 占位提示隐藏

- `TC02` `@` 唤醒菜单
  - 空输入下输入 `@`
  - 工具菜单出现
  - 首页下方提示词卡片集合保持不变

- `TC03` `@` 后直接回车
  - 仅输入 `@`
  - 直接按 `Enter`
  - 不自动吞首项
  - 不插入工具 token
  - 卡片集合保持不变

- `TC04` 键盘选择单个工具
  - 输入 `@`
  - `ArrowDown` 高亮
  - `Enter` 选中
  - 工具以内联 token 进入编辑区

- `TC05` 文本在前、数据源在后
  - 先输入 `hello `
  - 再输入 `@` 并选中工具
  - token 必须插在 `hello ` 后面

- `TC06` 单 token 删除
  - 在 `hello + token` 状态下
  - 按 `Backspace`
  - 删除紧邻 token，不误删普通文本

- `TC07` 多 token 混输
  - 输入 `hello `
  - 选中第一个工具
  - 再输入 `@a`
  - 选中第二个工具
  - 再继续输入 `s森森@`
  - 第二个 token 不得嵌进第一个 token 内部
  - 不得残留畸形字符，如半截 `epa`、孤立 `a`

- `TC08` 多 token 顺序稳定
  - 连续插入两个工具
  - token 顺序必须与选择顺序一致
  - 普通文本必须保留在原本位置

- `TC09` 多 token 后再次 `@`
  - 在 `hello + token1 + token2 + 中文` 的状态下
  - 末尾再次输入 `@`
  - mention 菜单应再次正常唤起

- `TC10` 多 token 节点序列
  - childNodes 应保持为：文本、token、空格文本、token、尾部文本
  - 不允许 token 嵌套 token

- `TC11` 失焦关闭
  - mention 菜单打开后点击输入框外部
  - 菜单关闭

- `TC12` 卡片区解耦
  - 输入框里选工具、删工具、输入普通文本
  - 首页下方提示词卡片不得变化

- `TC13` 浏览态独立
  - 点击首页能力胶囊，例如 `Keepa`
  - 卡片集合发生筛选变化
  - 之后再到输入框里用 `@` 选工具
  - 卡片仍保持胶囊筛选结果

- `TC14` mention 菜单层级
  - mention 菜单展开后
  - 下面的提示词卡片不得压到菜单上方
  - 菜单应位于输入框和首页卡片之上

- `TC15` 首页示例任务弹层
  - 点击第一张示例任务卡片
  - 弹层标题、`提示词(Prompt)`、`取消`、`查看回放`、`使用` 都正常出现
  - mention 状态不得污染弹层

- `TC16` 工程检查
  - `npm run lint` 通过
  - `npm run build` 通过

- `TC17` 控制台健康
  - 刷新后的当前会话里
  - 不应出现新的输入框运行时报错
